# 운영 테스트 가이드

테스트 주소: `https://bookstore-news-studio.rlawksml.chatgpt.site/`

실제 운영 데이터에 영향을 줄 수 있으므로 테스트 소식에는 `[테스트]`를 붙이고, 가능하면 다음 발행 월의 실제로 사용할 책방에서 검증한 뒤 올바른 내용으로 정리합니다. 입력자와 HTML 편집자를 동시에 확인할 때는 같은 브라우저의 일반 탭 두 개 대신 서로 다른 브라우저나 일반 창·시크릿 창을 사용합니다.

## 준비

- 환경 A: 소식 입력자용 데스크톱 브라우저
- 환경 B: HTML 편집자 또는 두 번째 입력자용 다른 브라우저
- 환경 C: 방문자용 휴대전화 또는 로그인하지 않은 시크릿 창
- 테스트할 책방 1곳과 발행 월 1개
- 20MB 이하 JPG·PNG 사진 2장 이상

## 1. 공개 방문자

1. 환경 C에서 주소를 열었을 때 별도 로그인 없이 달력이 보이는지 확인합니다.
2. 이전 달·다음 달 이동, 날짜 선택, 책방·소식 검색을 확인합니다.
3. 소식 제목을 눌러 상세 내용과 축소 사진이 모바일에서 빠르게 열리는지 확인합니다.
4. 상세 화면을 스크롤할 때 뒤쪽 화면이 함께 움직이지 않는지 확인합니다.
5. 소식 입력이나 HTML 편집 버튼을 누르면 암호 입력 화면이 나오는지 확인합니다.

## 2. 소식 입력자

1. 환경 A에서 `소식 입력`을 누르고 입력자 암호로 접속합니다.
2. 발행 월과 완료 진행률을 확인하고 책방 한 곳을 엽니다.
3. 소식 제목·상세 내용·날짜·장소를 입력하고 자동 저장 문구를 확인합니다.
4. 사진 여러 장을 드래그해서 추가하고 순서를 바꿉니다.
5. 제목이나 상세 내용을 비운 채 `입력 마무리`를 눌러 누락 위치로 이동하는지 확인합니다.
6. 내용을 채운 뒤 작성 내용 미리보기와 수동 임시 저장을 확인합니다.
7. `입력 마무리` 후 목록에서 `입력 완료` 상태와 진행률 변화를 확인합니다.
8. `완료 내용 공유하기`가 책방명과 소식 제목을 포함하는지 확인합니다.

## 3. 동시 편집

1. 환경 A에서 특정 책방·발행 월 편집 화면을 계속 열어둡니다.
2. 환경 B에서 입력자 암호로 접속해 같은 책방·같은 월을 엽니다.
3. 환경 B에 `다른 소식 입력자가 이 내용을 편집 중입니다` 안내가 표시되는지 확인합니다.
4. 환경 B에서 다른 책방을 열면 안내가 사라지는지 확인합니다.
5. 환경 A에서 책방 목록으로 돌아간 뒤 환경 B에서 최대 1분 안에 안내가 사라지는지 확인합니다.
6. 브라우저를 강제 종료한 경우에는 최대 3분 후 다음 작업자가 인계할 수 있는지 확인합니다.

## 4. HTML 편집자와 게시

1. 환경 B에서 로그아웃하고 `HTML 편집` 암호로 다시 접속합니다.
2. 입력 완료된 책방만 표시되는지 확인합니다.
3. 개별 HTML 미리보기·코드 복사·사진 ZIP·HTML과 사진 ZIP을 확인합니다.
4. 통합본에서 포함 소식을 선택하고 책방 순서를 바꿉니다.
5. HTML 편집자 두 명이 같은 개별 소식이나 통합본을 열었을 때 동시 작업 안내를 확인합니다.
6. 테스트 게시 URL을 저장하고 `게시 완료`를 누릅니다.
7. 환경 C 방문자 상세 화면에서 게시 링크가 열리는지 확인합니다.

## 5. 종료 확인

1. 완료된 소식을 입력자가 다시 수정하면 `작성 중` 또는 `재게시 필요`가 올바르게 표시되는지 확인합니다.
2. 입력자와 HTML 편집자 모두 로그아웃합니다.
3. 작업 탭을 닫았다가 다시 열었을 때 자동으로 작업 화면에 들어가지 않는지 확인합니다.
4. `[테스트]` 문구, 임시 게시 URL, 불필요한 테스트 사진을 실제 내용으로 교체하거나 제거합니다.

## 기대되는 보호 동작

- 로그인하지 않은 방문자의 저장·사진 원본 요청은 거부됩니다.
- 잘못된 월·파일 형식·20MB 초과 사진은 저장되지 않습니다.
- 다른 세션은 남의 편집 임대를 해제할 수 없습니다.
- 만료된 편집 상태는 다음 작업자가 인계합니다.
- 오래된 화면에서 저장하면 조용히 덮어쓰지 않고 최신 내용 재불러오기를 안내합니다.
