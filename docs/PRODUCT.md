# 제품 요구사항

## 제품 정의

동네책방 소식 스튜디오는 **동네책방 소식을 수집·저장하고 개별/통합 inline CSS HTML을 발행하는 웹 서비스**입니다. Word를 HTML로 자동 변환하는 도구가 아니라, Word만 작성할 줄 아는 입력자가 익숙한 방식으로 내용을 적으면 HTML 편집자가 바로 결과물을 가져가는 구조화된 작업 공간입니다.

## 사용자와 목표

- 정보 입력자: 전체 책방 기본정보와 월별 소식을 한 책방씩 작성하고 입력 완료 상태를 공유합니다.
- HTML 편집자: 입력 완료된 소식만 열어 개별·통합 HTML과 원본 사진을 받습니다.
- 방문자: 모바일 달력과 검색으로 작성 중·입력 완료·게시 완료 소식을 확인합니다.

작업 단계는 `작성 중 → 입력 완료 → 게시 완료` 세 단계뿐입니다. 검토·수정 요청·승인 단계와 작성자 ID는 두지 않습니다.

## 핵심 기능

- 발행 월 직접 선택, 책방별 한 화면 작성, 월별 진행률
- 책방 기본정보 저장과 반복 사용
- 제목·상세 내용 중심 입력과 필요할 때만 펼치는 선택 정보
- 이번 달 운영 안내, 여러 날짜, 사람이 읽는 일정 문구, 신청 방법
- 장소·참가비·마감일·표시 라벨, 자유로운 추가 항목, 여러 링크
- 여러 원본 사진 업로드, 모바일 미리보기 생성, 사진 드래그 정렬
- 자동 임시 저장과 수동 저장, 지난달 소식 복사, 작성 중 이탈 경고
- 개별 inline CSS HTML, 사진 ZIP, HTML+사진 ZIP
- 책방 순서와 포함 소식을 정하는 제목 중심 통합 HTML
- 모바일 달력, 디바운스 검색, 간결한 카드와 상세 화면

## 저장과 보안

- Database: Supabase에 책방·월별 소식·작업 상태 저장
- Storage: 비공개 원본 버킷과 공개 모바일 미리보기 버킷 분리
- 작업 암호: 브라우저 코드에 포함하지 않고 서버에서만 비교
- 작업 세션: HttpOnly 서명 쿠키와 탭 `sessionStorage`의 임의 세션 ID를 함께 확인
- Supabase Auth: 별도 사용자 계정이 필요하지 않아 사용하지 않음

## MVP 범위 제외

- 기존 `.docx` 자동 가져오기와 자유 형식 AI 분석
- 일반 사용자를 위한 HTML/CSS 편집기
- 검토·승인 워크플로우와 개인별 계정
- 게시판 자동 게시
- 고급 반복 일정 규칙

## 성공 기준

- 일반 입력자가 Word보다 복잡하다고 느끼지 않고 한 책방 소식을 완료할 수 있습니다.
- HTML 편집자가 입력 내용을 다시 정리하지 않고 개별·통합 HTML과 원본 사진을 받을 수 있습니다.
- 방문자는 원본 사진을 내려받지 않고 빠른 미리보기만 사용합니다.
- 작업 암호와 Supabase Secret Key가 브라우저 번들과 GitHub에 포함되지 않습니다.
