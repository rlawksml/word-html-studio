# 사용자 흐름

## 공통 진입

서비스 첫 화면은 모바일 방문자 달력입니다. 작업자는 헤더에서 `소식 입력` 또는 `HTML 편집`을 먼저 선택하고 해당 역할의 암호를 입력합니다. 새로고침해도 현재 탭에서는 역할이 유지되며 로그아웃하거나 탭을 닫으면 해제됩니다. 로그아웃하거나 왼쪽 위 로고로 방문자 메인에 이동하면 달력은 초기 월로, 선택 날짜·검색·정렬은 초기 상태로 돌아갑니다.

## 정보 입력자

1. 발행 월을 직접 선택합니다.
2. 책방 목록에서 한 곳을 선택하거나 기본정보를 등록합니다.
3. 제목과 상세 내용을 입력하고, 필요한 만큼 소식·날짜·사진을 추가합니다.
4. 선택 정보인 장소, 참가비, 신청 마감일, 신청 링크와 사진 설명을 필요할 때만 입력합니다.
5. 소식 카드를 끌어 표시 순서를 정합니다.
6. 자동 임시 저장을 이용하거나 `임시 저장`을 누릅니다. 필요하면 지난달 내용을 복사해 새 달 작성을 시작합니다.
7. 해당 책방의 입력이 끝나면 `입력 마무리`를 누릅니다. 별도 팝업 없이 책방 목록으로 돌아갑니다.
8. 이번 달에 포함할 모든 책방 입력을 마친 뒤 목록의 `완료 내용 공유하기`를 누릅니다. 완료된 책방·전체 소식 수와 책방별 소식 제목 목록이 담긴 월 전체 문구를 복사해 메신저로 전달합니다.
9. 완료 후 내용을 수정하면 자동으로 `작성 중`으로 돌아가며, 수정이 끝나면 다시 완료합니다.

## HTML 편집자

1. 발행 월을 선택합니다.
2. 입력 완료된 책방만 목록에서 확인합니다.
3. 개별 HTML을 미리보거나 코드를 복사합니다.
4. `사진 ZIP` 또는 `HTML + 사진 ZIP`을 내려받습니다. 사진 파일은 책방 이름과 순번으로 정리됩니다.
5. 통합본에서 포함할 소식을 선택하고 책방 순서를 끌어서 변경합니다.
6. 통합 HTML을 미리보고 복사합니다.
7. 외부 게시판에 게시한 뒤 선택적으로 게시 URL을 넣고 `게시 완료`를 누릅니다.
8. 게시 후 입력 자료가 바뀐 경우 `재게시 필요` 상태를 확인하고 다시 게시할 수 있습니다.

## 방문자

1. `이전 달`과 `다음 달`로 월을 이동합니다. 달력의 색상은 책방을 구분하며, 날짜에 마우스를 올리거나 초점을 두면 책방명과 소식 제목이 표시됩니다.
2. 날짜를 선택해 해당 날짜의 소식만 확인합니다. 선택한 날짜를 다시 누르면 이번 달 전체 소식으로 돌아갑니다.
3. 선택한 날짜의 책방 이름과 이벤트 제목을 간결한 모바일 카드로 봅니다. 게시 URL이 있으면 책방 이름을 눌러 게시글로 이동합니다.
4. 키워드로 책방·지역·소식을 검색합니다. 입력을 멈춘 뒤 검색 결과가 갱신되며 날짜순·최신순·책방순·소식 많은 순으로 정렬할 수 있습니다.

## 상태 규칙

```text
작성 중 → 입력 완료 → 게시 완료
   ↑          │           │
   └─ 완료 자료 수정       └─ 수정 시 재게시 필요
```

검토·수정 요청·승인 단계는 두지 않습니다. 입력자는 자신의 내용을 직접 수정하고 완료하며, HTML 편집자는 완료된 자료를 변환하고 게시 상태만 기록합니다.
