# 사용자 흐름

## 공통 진입과 세션

첫 화면은 모바일 방문자 달력입니다. 작업자는 헤더에서 `소식 입력` 또는 `HTML 편집`을 선택하고 역할별 공용 암호를 입력합니다. 암호는 서버에서만 확인하며 브라우저에는 저장하지 않습니다. 새로고침은 현재 탭의 역할을 유지하고, 탭을 닫거나 로그아웃하면 탭 세션 ID가 사라져 작업 API를 사용할 수 없습니다.

## 정보 입력자

상단에는 `① 발행 월 확인 → ② 소식 작성 → ③ 입력 마무리`가 항상 표시됩니다.

1. 이전·다음 달 버튼으로 발행 월을 선택하고 월별 완료 진행률을 확인합니다.
2. 목록에서 책방 한 곳을 선택합니다. 처음인 책방은 `책방 관리`에서 기본정보를 한 번 등록합니다.
3. 그 달만 달라지는 휴무·이전 안내가 있으면 `이번 달 운영 안내`를 작성합니다.
4. 소식 제목과 상세 내용을 Word처럼 작성합니다.
5. 달력에 필요한 날짜, 사람이 읽을 일정 안내, 사진을 필요한 만큼 추가합니다.
6. 장소·참가비·신청 방법·추가 항목·여러 링크가 필요하면 `추가 정보`를 펼칩니다.
7. 소식 카드와 사진을 드래그하거나 화살표 버튼으로 순서를 정합니다.
8. 입력은 1.2초 뒤 자동 저장되며 `임시 저장`으로 즉시 저장할 수도 있습니다.
9. 지난달 소식을 불러오면 제목과 본문은 복사되지만 날짜·일정·마감·신청 방법·신청 링크·참가비·사진은 비워집니다.
10. `입력 마무리`를 누르면 제목·상세 내용 누락 위치로 이동하거나, 이상이 없으면 책방 목록으로 돌아갑니다.
11. 모든 책방을 마치면 `완료 내용 공유하기`로 책방·소식 제목 목록을 복사해 메신저에 전달합니다.

완료된 내용을 수정하면 `작성 중`으로 돌아갑니다. 게시 완료 후 수정은 게시 상태를 지우지 않고 `재게시 필요`로 표시합니다.

## HTML 편집자

1. 발행 월을 선택하고 입력 완료된 책방만 확인합니다.
2. 개별 HTML 미리보기에서 운영 안내, 책방 링크, 일정·신청 방법·추가 항목이 올바른지 확인합니다.
3. HTML 코드를 복사하거나 `사진 ZIP`, `HTML + 사진 ZIP`을 받습니다. ZIP은 서버 세션을 확인한 뒤 비공개 원본 사진으로 만듭니다.
4. 통합본에서 포함할 소식을 선택하고 책방 순서를 드래그합니다.
5. 책방 이름·지역·소식 제목만 담긴 통합 HTML을 복사합니다.
6. 외부 게시 후 선택적으로 게시 URL을 저장하고 `게시 완료`를 누릅니다.

## 방문자

1. 이전·다음 달로 이동하고 책방별 색상 일정이 표시된 달력을 봅니다.
2. 날짜를 선택해 해당 날짜 소식만 확인하거나 검색어로 책방·지역·제목·내용을 찾습니다.
3. 카드에서는 책방 이름과 소식 제목만 간결하게 봅니다.
4. 소식 제목을 누르면 모바일 상세 화면에서 작업 상태, 본문, 일정, 신청 정보와 정렬된 미리보기 사진을 봅니다.
5. 게시 URL이 있으면 상세 화면에서 외부 게시글로 이동합니다.

## 상태 규칙

```text
작성 중 → 입력 완료 → 게시 완료
   ↑          │           │
   └─ 완료 자료 수정       └─ 수정 시 재게시 필요
```

검토·수정 요청·승인 단계는 두지 않습니다.
