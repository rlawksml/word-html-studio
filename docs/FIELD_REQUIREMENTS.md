# 입력 항목 필수·선택 분류

## 분석 기준

2026년 7월 Word 4개와 제공된 6월 개별 HTML·통합 HTML을 비교했습니다. 필수 여부는 “개별 HTML과 통합 HTML을 AI 없이 안정적으로 생성할 수 있는가”를 기준으로 정했습니다.

## 작성 화면 필수 항목

| 항목 | 사용처 | 필수인 이유 |
|---|---|---|
| 책방 이름 | 개별·통합 HTML 제목 | 모든 문서를 구분하는 기준 |
| 지역 | 통합 HTML 책방 제목 | 통합본의 `(울산 남구)` 같은 지역 표시 |
| 작성자 이름 | 검토·수정 요청 | HTML에는 출력하지 않지만 업무 담당자 확인에 필요 |
| 발행 월 | 개별·통합 HTML 제목 | `2026년 7월 소식` 생성 기준이며 시스템 기본값 제공 가능 |
| 소식 제목 | 개별 `<h2>`, 통합 `<strong>` | 모든 소식 블록의 식별자 |
| 내용 | 개별 HTML 본문 | Word에서 가장 핵심이 되는 원문 |
| 통합본용 한 줄 요약 | 통합 HTML 목록 | AI 없이 통합본을 생성하기 위해 작성자가 직접 제공 |

작성자가 실제로 새 소식마다 입력해야 하는 것은 `제목`, `내용`, `한 줄 요약` 세 가지입니다. 책방 정보와 발행 월은 계정 또는 이전 제출 값으로 자동 채우는 것이 운영 버전의 목표입니다.

## 선택 항목

| 항목 | 표시 조건 |
|---|---|
| 주소 | 입력된 경우 개별 HTML 책방 정보 상자에 표시 |
| 영업시간 | 입력된 경우 개별 HTML 책방 정보 상자에 표시 |
| 연락처 | 입력된 경우 개별 HTML 책방 정보 상자에 표시 |
| SNS·홈페이지 | 입력된 경우 개별 HTML 책방 정보 상자에 링크 또는 텍스트로 표시 |
| 분류 | 행사·독서모임·전시 등을 관리 화면에서 구분할 때 사용 |
| 진행 상태 | 모집 중·진행 중·종료 배지가 필요할 때 표시 |
| 시작일·종료일 | 일정이 있는 소식만 표시 |
| 장소 | 오프라인 장소나 Zoom 정보가 있을 때 표시 |
| 참가비 | 유료·무료 안내가 필요할 때 표시 |
| 신청 링크 | 신청이나 상세 페이지가 있을 때 버튼 표시 |
| 사진 | 없으면 이미지 영역 자체를 생성하지 않음 |

선택 항목은 빈 값일 때 라벨이나 빈 줄을 HTML에 남기지 않습니다.

## 사진 UX 규칙

- `사진 첨부하기` 버튼과 드래그 앤 드롭을 모두 제공
- 한 번에 여러 파일 선택 가능
- 첨부 직후 미리보기 목록 표시
- 개별 사진 삭제 가능
- 사용자가 선택한 순서대로 개별 HTML에 배치
- 사진은 선택 사항이며 0장부터 여러 장까지 허용
- 운영 버전에서는 파일 형식, 용량, 해상도 검증과 서버 업로드 필요

## Word·HTML에서 확인된 공통점

- 모든 Word에 책방 이름, 주소, 영업시간, 연락처와 SNS 계열 정보가 있었지만, 이는 소식마다 반복 입력할 필수값이 아니라 책방 프로필로 관리할 값입니다.
- 모든 소식에는 제목과 설명할 내용이 존재했습니다.
- 날짜, 장소, 참가비, 신청 방법은 소식 종류에 따라 없을 수 있었습니다.
- 네 Word 모두 이미지를 포함했지만, 텍스트만으로도 HTML이 성립하므로 이미지는 선택값으로 두는 것이 안전합니다.
- 통합 HTML은 개별 상세 내용을 모두 사용하지 않고 `책방 이름 + 지역 + 소식 제목 + 짧은 요약`을 사용했습니다.

## UX 원칙

1. 첫 화면에는 필수 항목만 노출합니다.
2. 날짜·장소·참가비 등은 `선택 정보 추가`를 눌렀을 때 펼칩니다.
3. 작성자는 Word처럼 긴 내용 칸에 문단을 자유롭게 작성합니다.
4. HTML 관련 용어는 작성자 화면에 노출하지 않습니다.
5. 필수값 누락은 제출할 때 한 번에 알리고, 작성 중에는 방해하지 않습니다.
