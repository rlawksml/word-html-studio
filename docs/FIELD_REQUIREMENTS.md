# 입력 항목 필수·선택 분류

## 분석 기준

2026년 7월 책방 Word 7개와 기존 개별·통합 HTML을 비교했습니다. 독서모임, 전시, 강연, 상품, 공간 대관처럼 형식이 다른 소식을 한 폼으로 처리하되 Word보다 입력 단계가 늘어나지 않는 것을 기준으로 삼았습니다.

## 필수 항목

| 범위 | 항목 | 사용처 |
|---|---|---|
| 책방 기본정보 | 책방 이름 | 개별·통합 HTML과 모든 화면의 식별자 |
| 책방 기본정보 | 지역 | 통합본과 방문자 검색 |
| 월별 작성 | 발행 월 | 월별 달력·HTML 제목·작업 상태 |
| 소식 | 소식 제목 | 방문자 카드·개별 HTML 제목·통합본 목록 |
| 소식 | 상세 내용 | 개별 HTML과 방문자 상세 본문 |

입력자가 매 소식마다 반드시 작성하는 항목은 **제목과 상세 내용 두 개**입니다. 날짜가 없는 굿즈·공간 대관·지원사업 소식도 있으므로 행사 날짜는 필수로 만들지 않습니다.

## 선택 항목

| 범위 | 항목 | 표시 조건 |
|---|---|---|
| 책방 | 주소·영업시간·소개 | 값이 있을 때 개별 HTML과 방문자 상세에 표시 |
| 책방 | 대표/추가 연락처 | 여러 담당 연락처가 있을 때 라벨과 함께 표시 |
| 책방 | 대표 SNS·홈페이지·추가 링크 | 유효한 URL만 링크로 표시 |
| 월별 | 이번 달 운영 안내 | 임시 휴무·이전·월 한정 영업시간이 있을 때 표시 |
| 소식 | 행사 날짜 여러 개 | 방문자 달력에 표시할 날짜 |
| 소식 | 일정 안내 | 시간·기간·반복 일정을 사람이 읽는 한 줄로 표시 |
| 소식 | 정기 여부 | `정기` 칩 표시 |
| 소식 | 표시 라벨 | 신청 중·마감·종료·신규 모집·상시 운영 중 선택 |
| 소식 | 신청 마감일·장소·참가비 | 입력된 값만 표시 |
| 소식 | 신청 방법 | DM·문자·전화·QR·댓글 등 링크가 아닌 방법 안내 |
| 소식 | 대표 신청 링크·추가 링크 | 유효한 URL만 표시 |
| 소식 | 자유로운 추가 항목 | 대상·정원·선정 도서·준비물 등 `항목명 + 내용` 반복 |
| 소식 | 사진·사진 설명 | 0장 이상, 설명은 사진별 선택 |

## 입력 화면 원칙

1. 제목·상세 내용·날짜·사진을 먼저 보여줍니다.
2. 장소·참가비·신청 방법·추가 항목·링크는 `추가 정보`를 펼쳤을 때만 보여줍니다.
3. 별도 시작·종료 시간 입력칸은 만들지 않고 `일정 안내`에 자연스럽게 작성합니다.
4. 상세 내용은 아직 서식 편집기 없이 Word처럼 문단을 작성하는 textarea를 사용합니다.
5. 빈 선택 항목은 HTML에 라벨이나 빈 줄을 생성하지 않습니다.
6. 통합본에는 상세 설명을 넣지 않고 `책방 이름 + 지역 + 선택된 소식 제목`만 사용합니다.

## 사진 UX와 저장 규칙

- 버튼 선택과 드래그 앤 드롭, 여러 파일 동시 첨부 지원
- 첨부 직후 모바일 미리보기 표시
- 드래그 또는 좌우 버튼으로 순서 변경
- 개별 삭제와 선택 사진 설명
- 원본은 비공개 다운로드용, 축소 JPEG는 방문자 공개용
- HTML과 ZIP의 사진 번호는 사용자가 정한 순서를 따름
