# Word 작성 가이드

이 가이드는 HTML이나 CSS를 전혀 모르는 사용자를 위한 문서 작성 규칙입니다. 핵심은 글자 크기나 색을 직접 꾸미는 대신 Word 상단의 **스타일**을 사용하는 것입니다.

> 기존 문서를 모두 고칠 필요는 없습니다. `[문서 제목]`, `<행사 제목>`, `내용:`, `일정:`처럼 일정한 표시를 사용한 문서도 프로그램이 읽습니다. Word 스타일을 함께 사용하면 오인식과 수동 확인을 더 줄일 수 있습니다.

## 1. 문서의 기본 구조

아래 순서로 작성하면 대부분의 템플릿에 안정적으로 변환됩니다.

```text
[제목] 문서 전체 제목

[부제] 선택 사항인 짧은 설명

[제목 1] 첫 번째 큰 항목
[본문] 설명 문단

[제목 2] 세부 항목
[본문] 설명 문단
[Word 목록 기능] 목록

[제목 1] 두 번째 큰 항목
[Word 표 기능] 표
```

대괄호 안의 말은 실제로 입력하지 않습니다. 해당 문장을 선택하고 Word의 **홈 → 스타일**에서 지정합니다.

## 2. 반드시 지켜야 할 규칙

### 문서 제목

- 문서 맨 위에 한 번만 작성합니다.
- 제목 줄에 Word의 `제목` 스타일을 적용합니다.
- 제목을 크게 보이게 하려고 글자 크기만 직접 키우지 않습니다.

### 큰 항목과 세부 항목

- 큰 항목에는 `제목 1` 스타일을 사용합니다.
- 그 아래 세부 항목에는 `제목 2` 스타일을 사용합니다.
- 더 작은 항목이 꼭 필요할 때만 `제목 3`을 사용합니다.
- 글자 크기, 굵기, 색상만 바꿔 제목처럼 만들지 않습니다.

### 본문

- 일반 설명에는 `표준` 또는 `본문` 스타일을 사용합니다.
- 문단을 나눌 때 Enter를 한 번 누릅니다.
- 간격을 만들기 위해 빈 줄을 여러 번 넣거나 Space를 반복하지 않습니다.

### 목록

- Word의 **글머리 기호** 또는 **번호 매기기** 기능을 사용합니다.
- `-`, `*`, `1)`를 키보드로 직접 입력해 목록처럼 만들지 않습니다.
- 하위 목록은 Tab 또는 목록 수준 변경 기능으로 만듭니다.

### 표

- 내용을 맞추기 위해 Space나 Tab을 반복하지 말고 Word의 **표 삽입** 기능을 사용합니다.
- 첫 번째 행은 열 제목으로 작성합니다.
- 셀 병합은 최소화합니다.
- 표 안에 또 다른 표를 넣지 않습니다.

### 이미지

- 복잡한 도형이나 텍스트 상자 대신 **그림 삽입**을 사용합니다.
- 이미지는 가능하면 `텍스트 줄 안` 배치로 설정합니다.
- 이미지 바로 아래 문단에 `그림 설명:`으로 시작하는 설명을 적습니다.
- 중요한 내용을 이미지 안의 글자로만 전달하지 않습니다.

### 링크

- 주소를 텍스트로만 붙이지 말고 Word의 **링크 삽입** 기능을 사용합니다.
- `여기`보다는 `신청 페이지 열기`처럼 목적이 드러나는 링크 문구를 씁니다.

## 3. 작성 예시

다음과 같은 공지문을 작성한다고 가정합니다.

```text
여름 교육 프로그램 안내                  ← 제목 스타일

신청 기간과 교육 일정을 안내합니다.       ← 부제 스타일(선택)

프로그램 개요                            ← 제목 1
이번 프로그램은 신규 사용자를 위한 교육입니다. ← 본문

교육 일정                                ← 제목 1
일시                                     ← 제목 2
2026년 8월 3일 오전 10시                 ← 본문

준비물                                   ← 제목 2
• 노트북                                 ← Word 글머리 기호
• 필기도구                               ← Word 글머리 기호

문의처                                   ← 제목 1
담당 부서와 연락처를 표로 작성            ← Word 표
```

변환기는 `제목 1`을 HTML의 큰 구역으로, `제목 2`를 하위 구역으로 인식합니다. 템플릿이 바뀌더라도 Word 원본을 다시 꾸밀 필요가 없습니다.

## 4. 피해야 하는 작성 방법

| 피해야 할 방법 | 대신 사용할 기능 |
|---|---|
| 글자 크기만 키워 제목처럼 만들기 | 제목 1, 제목 2 스타일 |
| Space를 여러 번 눌러 열 맞추기 | 표 삽입 |
| Enter를 여러 번 눌러 여백 만들기 | 문단 구분 한 번 |
| `-`를 직접 입력해 목록 만들기 | 글머리 기호 |
| 텍스트 상자 안에 본문 넣기 | 일반 문단 또는 표 |
| 머리글에 핵심 내용 넣기 | 본문 첫 부분에 작성 |
| 이미지 안에 설명을 전부 넣기 | 이미지 아래 설명 문단 추가 |

## 5. 업로드 전 체크리스트

- [ ] 파일 형식이 `.docx`인가요?
- [ ] 문서 맨 위에 `제목` 스타일이 적용된 제목이 하나 있나요?
- [ ] 큰 항목에 `제목 1`, 하위 항목에 `제목 2`를 사용했나요?
- [ ] 목록은 Word의 목록 기능으로 만들었나요?
- [ ] 열을 맞춰야 하는 내용은 표로 만들었나요?
- [ ] 이미지가 `텍스트 줄 안`으로 배치되어 있나요?
- [ ] 빈 줄과 Space를 반복해서 모양을 맞춘 곳이 없나요?
- [ ] 다른 사람이 읽어도 이해할 수 있는 파일명을 사용했나요?

## 6. 변환 결과에 경고가 표시될 때

- `문서 제목을 찾지 못했습니다`: 맨 위 제목에 `제목` 스타일을 적용합니다.
- `제목 단계가 건너뛰었습니다`: 제목 1 다음에 제목 3을 사용하지 않았는지 확인합니다.
- `직접 입력한 목록일 수 있습니다`: `-` 문자를 지우고 Word 글머리 기호를 적용합니다.
- `표가 너무 복잡합니다`: 병합된 셀이나 표 안의 표를 단순화합니다.
- `지원하지 않는 개체입니다`: 텍스트 상자, SmartArt 또는 도형 내용을 일반 문단이나 그림으로 바꿉니다.

## 관리자 참고

실제 배포 시에는 이 문서와 함께 올바른 스타일이 미리 들어간 `Word 작성 양식.docx`를 제공합니다. 사용자가 새 문서를 만드는 대신 양식을 복사해 작성하게 하면 오류를 크게 줄일 수 있습니다.

## 기존 작성 방식과 호환되는 간단 양식

Word 스타일 사용이 익숙하지 않은 조직은 아래 텍스트 규칙부터 도입할 수 있습니다.

```text
[지관서가 동네 책방 2026년 7월 소식 – 소담쓰담]

소담쓰담
- 주소: ...
- 영업시간: ...
- 연락처: ...
- 인스타그램: ...
- 방문 안내: ...

동네 책방 “소담쓰담”의 7월 소식입니다.

<2026 상반기 문학 독서모임 마무리>
- 테마: ...
- 내용: ...
[관련 이미지]

<7월 중국어 원서 독서모임>
- 선정도서: ...
- 내용: ...
- 모임 일정: ...
- 모임 방식: ...
[관련 이미지]
```

이 방식에서는 다음만 지키면 됩니다.

- 문서 전체 제목은 `[ ]`로 한 번만 표시합니다.
- 각 소식 제목은 `< >`로 표시합니다.
- 상세 내용은 `항목명: 내용` 형식으로 작성합니다.
- 이미지는 관련 소식의 상세 내용 뒤에 배치합니다.
- 항목명은 문서마다 바꾸지 않고 `주소`, `영업시간`, `연락처`, `인스타그램`, `내용`, `모임 일정`, `모임 방식`처럼 통일합니다.

이 간단 양식과 Word 제목 스타일은 함께 사용할 수 있습니다. 최종 권장안은 `<행사 제목>` 문단에 `제목 1` 스타일도 적용하는 것입니다.
