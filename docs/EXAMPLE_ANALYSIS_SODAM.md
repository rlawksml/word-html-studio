# 소담쓰담 Word 예시 분석

## 결론

이 Word 파일은 그대로 업로드 입력으로 사용할 수 있습니다. 사용자가 JSON을 작성할 필요는 없습니다. 변환기는 문서의 텍스트, 목록, 이미지와 순서를 읽어 내부 JSON을 자동 생성합니다.

## 확인된 문서 구조

- 문단 33개
- 표 0개
- 포함 이미지 3개
- 사용 문단 스타일: `Normal` 17개, `List Paragraph` 16개
- Word의 `제목`, `제목 1`, `제목 2` 스타일은 사용하지 않음
- 이미지가 inline shape가 아닌 floating drawing으로 저장됨

현재 문서의 의미 구조는 Word 스타일보다 텍스트 표기에 담겨 있습니다.

| Word 표기 | 해석 |
|---|---|
| `[지관서가 동네 책방 ...]` | 문서 전체 제목 |
| `소담쓰담` | 책방 이름 |
| `주소:`, `영업시간:`, `연락처:` | 책방 기본 정보 |
| `<2026 상반기 ...>` | 개별 소식 제목 |
| `테마:`, `내용:`, `모임 일정:` | 소식 속성 |
| 문단 사이 이미지 | 앞뒤 소식에 연결된 이미지 |

## 프로그램 내부 변환 예시

## 개별 HTML과의 매핑

- 문서 제목 → 최상단 `<h1>`
- 책방 기본 정보 → 배경색이 있는 정보 상자
- 소식 제목 → `<h2>`
- 테마/선정도서 → 강조 카드 또는 `<h3>`
- 내용 → 본문 `<p>`
- 일정/방식/문의 → `<ul><li>` 또는 정보 목록
- Word 이미지 → 해당 소식 블록 안의 `<img>`

제공된 6월 개별 HTML은 일부 문구와 상태를 새롭게 요약·각색하고 있습니다. 따라서 7월 Word를 HTML로 만들 때는 다음 두 모드를 구분해야 합니다.

- 충실 변환: Word 내용을 빠짐없이 구조화하고 템플릿에 배치
- 편집 변환: LLM이 제목, 요약, 배지, 이모지와 강조 문구를 생성

기본값은 충실 변환으로 하고, 편집 변환은 사용자가 미리보기에서 선택하도록 하는 것이 안전합니다.

## 통합 HTML 생성 규칙

통합 HTML은 개별 HTML 전체를 단순히 이어 붙이는 결과가 아닙니다. 각 문서에서 다음만 추출해 별도로 요약합니다.

- 책방 이름과 지역
- 주요 소식 1~3개
- 각 소식의 상태: 예정, 진행 중, 종료
- 날짜와 핵심 참여 정보

상태가 Word에 명확히 없으면 프로그램이 임의 확정하지 않고 `확인 필요`로 표시하거나 LLM 제안을 사용자에게 보여줍니다.

## 이 사례에서 필요한 보완

- `인스타그램 (@minxi1228)`을 실제 링크로 바꾸기 위한 URL 규칙
- 이미지가 floating drawing이어도 문단 순서를 기준으로 섹션에 귀속하는 처리
- 7월 문서 안의 `2026.09.05`가 오타인지 실제 일정인지 사용자 확인
- 통합본에 표시할 지역 정보를 책방 프로필에서 관리할지 Word에서 받을지 결정
- 외부 구분선 이미지처럼 Word에 없는 장식 자산을 템플릿 자산으로 관리
